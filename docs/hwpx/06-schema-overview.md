# 6 OWPML 스키마 구성

이 표준에서 기술하고 있는 OWPML 문서의 논리적 구조를 정의하는 XML 스키마는 5개로 구성된다.

- **첫째**는 문서 내에서 사용되는 모든 설정들을 가지고 있는 header 스키마이다. header에는 '구역'에 대한 정의, 글꼴 문자열, 글자 설정/문단 설정과 같은 스타일 정보, 외부 참조 객체에 대한 연결 정보 등이 들어있다.

- **둘째**는 실제 문서 내용이 들어있는 body 스키마이다. body에는 사용자가 설정한 문서의 구조 대로 문서의 내용들이 저장된다.

- **셋째**는 문서에 공통으로 적용되는 바탕 쪽 설정이 들어있는 스키마이다.

- **넷째**는 문서 이력 스키마이다. 문서 이력 정보는 추가/수정되었던 기록을 담고 있다.

- **다섯째**는 포맷에 대한 버전 스키마이다. 포맷의 버전에 대한 기록을 담고 있다.

각 XML 스키마에 대한 주요 내용 설명은 다음 절부터 진행한다. 각 스키마에 대한 상세한 스키마 정의는 첨부된 [부록B] XML 스키마 문서 5종을 참조한다.
