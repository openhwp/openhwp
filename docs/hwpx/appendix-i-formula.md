# 부속서 I (규정) 수식 스크립트

## I.1 수식 버전

이 문서를 기준으로 수식 스크립트의 최신 버전은 6.0이다.

OWPML 본문 내에서는 수식 버전을 다음과 같이 표기한다.

```
Equation Version 60
```

## I.2 항의 구분 및 인식

수식에서 항(토큰)의 구분은 공백(Space, Tab)이나 줄바꿈으로 한다. 만약 공백을 포함한 여러 항을 하나의 항으로 인식하기 위해서는 { }를 사용해야 한다.

| 입력 | e ^ {2x^2 + y^2} |
| ---- | ---------------- |
| 결과 | $e^{2x^2 + y^2}$ |

수식에서 한 항이 9자를 넘어서면 수식 스크립트 엔진은 이를 2개의 항으로 분리하여 인식한다. 9자 이상의 항을 하나의 항으로 처리하기 위해서는 따옴표("")를 '항의 앞뒤에' 사용해야 한다.

| 입력 | 1234567890 over 5       |
| ---- | ----------------------- |
| 결과 | $\frac{123456789^0}{5}$ |

| 입력 | "1234567890" over 5    |
| ---- | ---------------------- |
| 결과 | $\frac{1234567890}{5}$ |

## I.3 수식에 쓰이는 글씨체

기본적으로 수식 편집기에서 입력하는 로마자들은 이탤릭체로 바뀌어 표현된다. 특별히 로만(영문-수식)체를 쓸 때에는 로만체 전환 명령어인 "rm"을 앞세워야 한다. 또 볼드체를 입력하기 위해서는 명령어 "bold"를 앞세운다.

| 기본 | 이탤릭체                       |
| ---- | ------------------------------ |
| rm   | 로만체                         |
| it   | 로만체 입력 중 이탤릭체로 변경 |
| bold | 볼드체                         |

### I.3.1 예제

#### I.3.1.1 기본

| 입력 | Equation font test  |
| ---- | ------------------- |
| 결과 | _Equation fonttest_ |

#### I.3.1.2 로만체 → 이탤릭체

| 입력 | Equation rm font it test |
| ---- | ------------------------ |
| 결과 | _Equation_ font*test*    |

#### I.3.1.3 볼드체

| 입력 | rm Equation ~ bold Editor |
| ---- | ------------------------- |
| 결과 | Equation **Editor**       |

## I.4 White Space 표현

### I.4.1 공백 및 줄바꿈

수식 스크립트 엔진은, 공백(Space)과 줄바꿈 등은 항을 구분하는 역할로만 쓰이며 화면 표시에는 사용되지 않고 무시된다. 수식 결과에 '공백을 표시하기' 위해서는 '~'나 '`'를 사용해야 한다. 따옴표("")는 여러 개의 공백이 들어간 항을 표현할 때에도 사용될 수 있다. 줄바꿈은 #을 사용해서 표현할 수 있다.

| 입력 | "수식~편집기"`"예제" |
| ---- | -------------------- |
| 결과 | 수식 편집기 예제     |

| 입력 | "2차 방정식의 해법에는 인수분해에 의한"# "해법과 근의 공식에 의한 해법이 있다." |
| ---- | ------------------------------------------------------------------------------- |
| 결과 | 2차 방정식의 해법에는 인수분해에 의한<br>해법과 근의 공식에 의한 해법이 있다.   |

### I.4.2 세로 칸 맞춤(탭)

세로 칸 맞춤은 맞추고자 하는 기준 글자 앞에 "&"를 입력해 주어야 한다. 즉 글자 앞에서 `<Tab>`을 누른 것과 같은 효과를 준다.

| 입력 | 420 = 2 times 210 #<br>= 2 times 2 times 105 #<br>= 2^2 times 3 times 35 #<br>= 2^2 times 3 times 5 times 7                                                     |
| ---- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 결과 | $420 = 2 \times 210$<br>$\phantom{420} = 2 \times 2 \times 105$<br>$\phantom{420} = 2^2 \times 3 \times 35$<br>$\phantom{420} = 2^2 \times 3 \times 5 \times 7$ |

| 입력 | 420 = & 2 times 210 #<br>= & 2 times 2 times 105 #<br>= & 2^2 times 3 times 35 #<br>= & 2^2 times 3 times 5 times 7   |
| ---- | --------------------------------------------------------------------------------------------------------------------- |
| 결과 | $420 = 2 \times 210$<br>$= 2 \times 2 \times 105$<br>$= 2^2 \times 3 \times 35$<br>$= 2^2 \times 3 \times 5 \times 7$ |

## I.5 예약어 및 명령어

예약어 및 명령어는 몇몇 예약어를 제외하고 기본적으로 대소문자를 구분하지 않는다. 대소문자를 구분하는 예약어인 경우 설명에 이를 명시한다.

### I.5.1 따옴표("")

글자 길이가 긴 단어나 공백을 포함한 문장을 하나로 묶기 위해 사용되는 예약어

### I.5.2 공백 및 줄바꿈

| 예약어 | 설명                    |
| ------ | ----------------------- |
| ~      | 보통 길이의 공백을 지정 |
| `      | 1/4 길이의 공백을 지정  |
| #      | 줄바꿈을 지정           |

### I.5.3 무한대(INF)

| 입력 | INF |
| ---- | --- |
| 결과 | ∞   |

### I.5.4 곱셈(TIMES)

| 입력 | TIMES |
| ---- | ----- |
| 결과 | ×     |

### I.5.5 분수(OVER)

분수는 기본적으로 가운데 맞춤으로 정렬한다. 오른쪽이나 왼쪽 맞춤을 쓰려면 빈칸 넣기인 `<->`나 `<>`를 이용한다.

| 입력 | 1 OVER 1 000 + (1~~~) OVER 1 000 - (~~~~1) OVER 1 000                        |
| ---- | ---------------------------------------------------------------------------- |
| 결과 | $\frac{1}{1000} + \frac{1\phantom{000}}{1000} - \frac{\phantom{000}1}{1000}$ |

### I.5.6 위 아래(ATOP)

"OVER" 명령과 같지만 분수를 나타내는 가로선을 생략해 준다.

| 입력 | x ATOP y    |
| ---- | ----------- |
| 결과 | $x \atop y$ |

### I.5.7 지수(^, SUP)

| 입력 | x ^ y |
| ---- | ----- |
| 결과 | $x^y$ |

### I.5.8 첨수 표기 (\_, SUB)

| 입력 | x \_ 8 |
| ---- | ------ |
| 결과 | $x_8$  |

### I.5.9 제곱근(SQRT)

"SQRT"를 대신하여 "ROOT"라고 입력해도 같은 결과를 얻을 수 있다.

| 입력 | x SQRT (y + z) |
| ---- | -------------- |
| 결과 | $x\sqrt{y+z}$  |

| 입력 | SQRT (3) of (y + z) |
| ---- | ------------------- |
| 결과 | $\sqrt[3]{y+z}$     |

### I.5.10 LEFT, RIGHT

LEFT와 RIGHT가 양쪽에서 하나의 쌍을 이루도록 한다. 만약 한쪽을 생략하고 싶을 경우 생략하려는 쪽에 마침표(.)을 넣는다.

| 입력 | LEFT(a+b over x+y RIGHT)       |
| ---- | ------------------------------ |
| 결과 | $\left(\frac{a+b}{x+y}\right)$ |

| 입력 | LEFT . a+b over x+y RIGHT }     |
| ---- | ------------------------------- |
| 결과 | $\left.\frac{a+b}{x+y}\right\}$ |

### I.5.11 BIGG 기호

| 입력 | LEFT { a+b OVER a-b BIGG / x+y OVER x-y RIGHT }        |
| ---- | ------------------------------------------------------ |
| 결과 | $\left\{\frac{a+b}{a-b} \Big/ \frac{x+y}{x-y}\right\}$ |

### I.5.12 적분(INT, OINT)

| 입력 | INT \_0^inf 1 over x dx       |
| ---- | ----------------------------- |
| 결과 | $\int_0^\infty \frac{1}{x}dx$ |

| 입력 | OINT \_C_1 1 over x dx      |
| ---- | --------------------------- |
| 결과 | $\oint_{C_1} \frac{1}{x}dx$ |

### I.5.13 극한(lim, Lim)

본 예약어는 대소문자를 구분하므로, 반드시 대소문자를 정확하게 표기해야 한다.

| 입력 | lim \_{x->0} {1 over x}      |
| ---- | ---------------------------- |
| 결과 | $\lim_{x \to 0} \frac{1}{x}$ |

| 입력 | Lim \_{x->0} {1 over x}            |
| ---- | ---------------------------------- |
| 결과 | $\text{Lim}_{x \to 0} \frac{1}{x}$ |

### I.5.14 합(SUM)

| 입력 | x = SUM \_{i=0} ^{inf} {x sub i } |
| ---- | --------------------------------- |
| 결과 | $x = \sum_{i=0}^{\infty} x_i$     |

### I.5.15 합집합(UNION), 교집합(INTER), 곱집합(PROD)

| 입력 | A UNION B  |
| ---- | ---------- |
| 결과 | $A \cup B$ |

| 입력 | A INTER B  |
| ---- | ---------- |
| 결과 | $A \cap B$ |

| 입력 | A PROD B    |
| ---- | ----------- |
| 결과 | $A \prod B$ |

### I.5.16 상호관계(REL)

두 항 간의 상호관계를 상세히 표현할 수 있는 기능으로 두 항을 연결하는 화살표의 위/아래에 관계식 등의 내용을 삽입할 수 있다.

| 입력 | A REL Lrarrow abc xyz B     |
| ---- | --------------------------- |
| 결과 | $A \xleftarrow[xyz]{abc} B$ |

| 입력 | rm PbO_2 + 2H_2 SO_4 + Pb<br>REL exarrow 충전 방전<br>2PbSO_4 + 2H_2 O                                                                 |
| ---- | -------------------------------------------------------------------------------------------------------------------------------------- |
| 결과 | $\text{PbO}_2 + 2\text{H}_2\text{SO}_4 + \text{Pb} \xrightleftharpoons[\text{방전}]{\text{충전}} 2\text{PbSO}_4 + 2\text{H}_2\text{O}$ |

### I.5.17 BUILDREL

Relation 기능과 유사한 기능으로 화살표 아래 부분의 내용을 생략할 수 있다.

| 입력 | A BUILDREL Lrarrow { f(a) + g(a) + h(a) } B |
| ---- | ------------------------------------------- |
| 결과 | $A \xleftarrow{f(a) + g(a) + h(a)} B$       |

| 입력 | rm RCH_2 CH_2 COSCoA<br>BUILDREL rarrow -2H~<br>RCH=CHCOSCoA                               |
| ---- | ------------------------------------------------------------------------------------------ |
| 결과 | $\text{RCH}_2\text{CH}_2\text{COSCoA} \xrightarrow{-2\text{H}} \text{RCH}=\text{CHCHSCoA}$ |

### I.5.18 세로 쌓기(PILE, LPILE, RPILE)

위에 있는 문자를 기준으로 가운데(PILE), 왼쪽(LPILE), 오른쪽(RPILE) 맞춤을 선택할 수 있다.

| 입력 | PILE { t^3 # t^2 # t }                        |
| ---- | --------------------------------------------- |
| 결과 | $\begin{matrix} t^3 \\ t^2 \\ t \end{matrix}$ |

### I.5.19 경우들(CASES)

여러 개의 행 전체를 나타내는 묶음표({)는 상황에 따라 그 크기가 자동으로 확대되어 표시된다.

| 입력 | sign(x) = CASES{ 1 & x>0 #<br>0 & x=0 #<br>-1 & x=0 }                      |
| ---- | -------------------------------------------------------------------------- |
| 결과 | $sign(x) = \begin{cases} 1 & x > 0 \\ 0 & x = 0 \\ -1 & x = 0 \end{cases}$ |

| 입력 | BINOM n x      |
| ---- | -------------- |
| 결과 | $\binom{n}{x}$ |

### I.5.21 행렬(MATRIX, PMATIRX, BMATRIX, DMATRIX)

matrix는 줄(행) 단위로 입력하는 일반적인 방법과, matrix 다음에 col을 입력하여 칸(열) 단위로 입력하는 방법이 있다.

col을 넣어 행렬을 칸 단위로 입력할 때는 각 수식 앞에 col(가운데) 대신 lcol(왼쪽 맞춤), rcol(오른쪽 맞춤)을 넣어 정렬 위치를 지정할 수도 있다.

matrix 대신에 pmatrix를 쓰면 소괄호 ( ), bmatrix를 쓰면 대괄호 [ ], dmatrix를 쓰면 세로줄 |을 행렬(matrix) 양쪽에 각각 표시해 준다.

| 입력 | matrix {<br>a_1 & b_1 & c_1 & d_1 #<br>a_2 & b_2 & c_2 & d_2 #<br>a_3 & b_3 & c_3 & d_3 #<br>a_4 & b_4 & c_4 & d_4<br>}        |
| ---- | ------------------------------------------------------------------------------------------------------------------------------ |
| 결과 | $\begin{matrix} a_1 & b_1 & c_1 & d_1 \\ a_2 & b_2 & c_2 & d_2 \\ a_3 & b_3 & c_3 & d_3 \\ a_4 & b_4 & c_4 & d_4 \end{matrix}$ |

| 입력 | matrix {<br>lcol { a_1 # abc_2 # a_3 # a_4 }<br>rcol { abc_1 # b_2 # b_3 # b_4 }<br>col { c_1 # c_2 # c_3 # c_4 }<br>col { d_1 # d_2 # d_3 # d_4 }<br>} |
| ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 결과 | $\begin{matrix} a_1 & abc_1c_1d_1 \\ abc_2 & b_2c_2d_2 \\ a_3 & b_3c_3d_3 \\ a_4 & b_4c_4d_4 \end{matrix}$                                              |

| MATRIX                                                                              | PMATIRX                                                                               | BMATRIX                                                                               | DMATIRX                                                                               |
| ----------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| $\begin{matrix} a_1 & b_1 & c_1 \\ a_2 & b_2 & c_2 \\ a_3 & b_3 & c_3 \end{matrix}$ | $\begin{pmatrix} a_1 & b_1 & c_1 \\ a_2 & b_2 & c_2 \\ a_3 & b_3 & c_3 \end{pmatrix}$ | $\begin{bmatrix} a_1 & b_1 & c_1 \\ a_2 & b_2 & c_2 \\ a_3 & b_3 & c_3 \end{bmatrix}$ | $\begin{vmatrix} a_1 & b_1 & c_1 \\ a_2 & b_2 & c_2 \\ a_3 & b_3 & c_3 \end{vmatrix}$ |

### I.5.25 최소공배수/최대공약수 함수(LADDER)

| 입력 | SLADDER{ 2&12& # 2&6&0 # 2&3&0 # 1&1<br>&} |
| ---- | ------------------------------------------ | ------------------------------------------------------------------------- |
| 결과 | $\begin{array}{c                           | c} 2 & 12 \\ 2 & 6 & \cdots 0 \\ 2 & 3 & \cdots 0 \\ & 1 & 1 \end{array}$ |

### I.5.27 거꾸로 나눗셈(LONGDIV)

나눗셈을 표현할 때 사용하는 명령어이다.

Layout을 맞추기 위한 방법으로는 숫자만 입력해서 자동으로 Layout을 맞추거나 '~'를 이용해서 명시적으로 Layout을 맞출 수 있다. 마지막 인자에 숫자가 아닌 문자가 들어가면 자동 Layout을 하지 않는다.

| 입력 | LONGDIV {6} {422} {2532#24#13#12#12#12#0}<br>LONGDIV {6} {422} {2532#24#~13#~12#~~12#~~12#~~~0}                  |
| ---- | ---------------------------------------------------------------------------------------------------------------- |
| 결과 | $\begin{array}{r} 422 \\ 6 \overline{)2532} \\ 24 \\ \hline 13 \\ 12 \\ \hline 12 \\ 12 \\ \hline 0 \end{array}$ |
