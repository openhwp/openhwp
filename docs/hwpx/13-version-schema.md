# 13 파일 형식 버전 정보 XML 스키마

## 13.1 파일 형식 버전 정보

여러가지 요인에 의해서 문서 형식의 구조 또는 세부 속성들이 변경될 수 있다. 말단 요소에 속성을 추가하는 경우 하위 버전의 문서 형식과 호환성을 가질 수 있다. 하지만 대규모의 구조 변경 또는 기존 요소의 삭제와 같은 경우에는 하위 버전의 문서 형식과 호환성을 가질 수 없게 된다. 이를 위해서 이 표준에서 정의한 문서 형식에서는 문서 형식의 버전을 지정할 수 있는 부분을 포함하고 있다. 이 장에서는 문서의 파일 형식 버전 정보 스키마에 대해서 기술한다.

## 13.2 version 요소

문서의 파일 형식 버전 정보를 표현하기 위한 요소이다.

이 표준에서는 워드프로세서 문서에 한해서 기술하고 있다. 하지만, 이 표준에서 논의하고 있는 일부 요소들은 워드프로세서 문서뿐만 아니라 다른 응용프로그램의 문서 규격에서도 사용될 수 있다. `<version>` 요소에서는 이를 위해서 속성 `@targetApplication`을 두어서 대상 응용 프로그램을 지정할 수 있다.

### 표 312 — version 요소

| 속성 이름         | 설명                                                                                                                                 |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| targetApplication | 오피스 문서 형식 중의 하나만 표현함.<br/>(WORDPROCESSOR, PRESENTATION, SPREADSHEET)                                                  |
| major             | 문서 형식의 구조가 완전히 바뀌는 것을 나타냄.<br/>숫자가 다르면 구버전과 호환 불가능                                                 |
| minor             | 큰 구조는 동일하나, 큰 변화가 있는 것을 나타냄.<br/>숫자가 다르면 구버전과 호환 불가능                                               |
| micro             | 구조는 동일하나 요소가 추가되었거나, 하위 버전에서<br/>호환되지 않는 정보가 추가된 것을 나타냄.<br/>숫자가 달라도 구버전과 호환 가능 |
| buildNumber       | 요소에 정보들이 추가된 것을 나타냄 숫자가 달라도<br/>구버전과 호환 가능                                                              |
